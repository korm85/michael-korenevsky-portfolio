# Prompt for responding to PM portfolio feedback

You have memory about Michael Korenevsky's PM work, career, decisions, metrics, and voice. Use it.

Before writing anything, read `docs/product-case-studies.md`. It contains the verified technical specs, commercial metrics, named case studies (Baker Hughes, Knauf, Emerson), competitive comparisons, and roadmap decisions for both AMVero and the Simulation Suite. These are the facts behind every metric on the site — use them to write copy that is grounded in specific, real data, not generalisations.

The file `docs/pm-portfolio-review.md` contains feedback from a Head of PM who reviewed Michael's portfolio. It has 8 items (FB-01 through FB-08). Each item describes a specific piece of copy that either reads as generic or as product marketing instead of PM achievement.

For each feedback item, write the revised copy that should replace it. Draw from `docs/product-case-studies.md` and your memory of Michael's career — his real decisions, real trade-offs, real customers, real career arc — to make the copy specific to him. Do not write generic PM language.

For each item respond with:

**FB-XX**
- Current: [the existing copy]
- Revised: [your replacement]

Keep revised copy the same length as the current copy unless the feedback explicitly calls for expansion.

**Audience note:** The portfolio targets PMs and hiring managers across industries — not 3D printing specialists. Replace domain-specific technical terms with language that conveys the same PM signal to a general audience. For example: "recoater line detection" → "defect detection mid-build"; "thermo-mechanical solver" → "physics-based simulation"; "LPBF anomaly types" → "manufacturing defects". The goal is for any senior PM to immediately understand the value of the decision, not to demonstrate industry depth.
